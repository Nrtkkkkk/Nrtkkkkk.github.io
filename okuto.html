githubの使い方
ファイルの状態を確認git status
ファイルを確認!git add hello.html
ローカルリポジトリを作成するgit init
ローカルリポジトリのファイルをGitHubのリポジトリに送信git push origin master
ローカルリポジトリにインデックスに追加したファイルを登録git commit -m

githubpageの作り方

リポジトリの作成　(プロジェクト)

1.ターミナルで下記を実行、ローカルのリポジトリを指定
$ cd /path/to/local-repository

2.【リポジトリネーム】のブランチを制作
$ git branch 【リポジトリネーム】

3.チェックアウトする
$ git checkout 【リポジトリネーム】

リポジトリの作成　(ユーザー)

$ git clone https://github.com/【GitHubアカウント名】/【GitHubアカウント名】.github.io 実行することでローカルにファイルが生成されています。 ファイル名：【GitHubアカウント名】.github.io 実行の指定先を移動します。 $ cd 【GitHubアカウント名】

4.htmlを作成

ターミナルで下記を実行し、ローカルにhtmlファイルを生成します。 $ echo "Hello world" > index.html 下記を実行して、ファイルをGitHubにコミットします。 $ git add --all $ git commit -m "First commit"

5.GitHubへPush

ブランチをプッシュ （プロジェクト） $ git push -u origin【リポジトリネーム】 （ユーザー） $ git push -u origin master

6.確認

プロジェクト http://【ユーザーネーム】.github.io/【リポジトリネーム】 ユーザー http://【ユーザーネーム】.github.io

githubpageの更新の仕方

$ cd /path/to/local-repository $ echo "Hello GitHub Pages!" > index.html

ファイルの更新が終わったら、下記のように入力して変更をコミットしてください。 $ git add --all $ git commit -m "Update page"

先ほどと同じように、ユーザサイトならば「master」ブランチをプッシュしてください。 $ git push -u origin master

プロジェクトサイトならば「gh-pages」ブランチをプッシュします。 $ git push -u origin gh-pages
